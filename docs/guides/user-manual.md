## 1. 系统简介

### 1.1 什么是 ERP 查询智能体

ERP 查询智能体是一个基于自然语言处理的智能查询系统，允许用户使用日常语言描述来查询 ERP 数据，系统会自动将自然语言转换为 SQL 语句执行，并返回结构化的查询结果。

### 1.2 主要功能

- **AI 智能查询**: 用自然语言描述您的数据需求，系统自动生成 SQL 并执行
- **多数据源支持**: 支持连接 MySQL 和 SQL Server 等多种数据库
- **灵活的映射配置**: 将外部数据库表字段映射为业务友好的别名
- **权限管理系统**: 基于角色的访问控制 (RBAC)
- **可视化报表**: 查询结果支持多种图表展示

## 3. AI 智能查询

AI 智能查询是系统的核心功能，让您用自然语言查询 ERP 数据。

### 3.1 基本查询流程

1. 点击左侧菜单「AI查询」进入查询页面
2. 在文本框中输入您的查询需求
3. 点击「查询」按钮或按回车
4. 等待系统处理（包含 AI 分析、SQL 生成、执行查询）
5. 查看返回的结构化结果

### 3.2 查询示例

**示例 1: 统计查询**
```
输入: 统计2024年每个月的销售总额
```

**示例 2: 筛选查询**
```
输入: 查询华北地区2024年Q1的客户订单，按金额降序排列
```

**示例 3: 对比分析**
```
输入: 对比2024年上半年与下半年的月度采购金额
```

### 3.3 查询结果展示

查询完成后，系统提供多种展示方式：

- **表格视图**: 分页展示原始数据
- **图表视图**: 支持柱状图、折线图、饼图、雷达图等
- **导出功能**: 支持导出为 PDF 文件

### 3.4 配置轴映射

在图表视图中，您可以：

1. 点击「配置轴」打开轴选择器
2. 选择 X 轴字段（如：时间、类别）
3. 选择 Y 轴字段（如：金额、数量）
4. 选择图表类型
5. 点击「应用」查看配置后的图表

## 4. 数据源管理

数据源是连接外部 ERP 数据库的桥梁。

### 4.1 添加数据源

1. 进入「数据源」→「数据源列表」
2. 点击「新增数据源」按钮
3. 填写以下信息：
   - **名称**: 数据源显示名称（如：生产数据库）
   - **类型**: MySQL 或 SQL Server
   - **主机**: 数据库服务器地址
   - **端口**: 数据库端口（MySQL 默认 3306）
   - **数据库名**: 要连接的数据库名称
   - **用户名**: 数据库访问用户名
   - **密码**: 数据库访问密码
4. 点击「测试连接」验证连接是否成功
5. 点击「保存」完成创建

### 4.2 编辑数据源

1. 在数据源列表中找到要编辑的数据源
2. 点击操作列的「编辑」按钮
3. 修改需要更新的信息
4. 点击「保存」

> **注意**: 密码字段默认为空，修改时只填入新密码，不填则保持原密码不变。

### 4.3 删除数据源

1. 在数据源列表中找到要删除的数据源
2. 点击操作列的「删除」按钮
3. 在确认弹窗中点击「确定」

> **警告**: 删除数据源将同时删除关联的表映射和字段映射，请谨慎操作。

### 4.4 获取表和字段信息

1. 点击数据源操作列的「Tables」按钮
2. 在弹窗中查看该数据源下的所有表
3. 点击表名可查看该表的所有字段信息

## 5. 表与字段映射

通过映射配置，将数据库表字段转换为业务友好的描述。

### 5.1 表映射管理

表映射用于将外部数据库表映射为业务语义名称。

**创建表映射:**
1. 进入「数据源」→「表映射」
2. 选择目标数据源
3. 点击「新增表映射」
4. 填写：
   - **源表名**: 外部数据库中的实际表名
   - **别名**: 业务友好的显示名称（如：客户订单表）
   - **查询规则**: 额外的 SQL 条件（如：`is_delete = 0`）
5. 点击「保存」

### 5.2 字段映射管理

字段映射定义表中每个字段的业务含义。

**创建字段映射:**
1. 进入「数据源」→「表映射」
2. 找到要配置字段的表映射，点击「字段配置」
3. 点击「新增字段映射」
4. 填写：
   - **源字段名**: 数据库中的实际字段名
   - **别名**: 业务友好的显示名称（如：订单编号）
   - **描述**: 字段用途说明（帮助 AI 理解）
   - **显示规则**: 值转换规则，格式为 JSON
     - 例如：`{"0": "未支付", "1": "已支付", "2": "已取消"}`
5. 点击「保存」

### 5.3 字段映射说明

| 字段 | 说明 | 示例 |
|------|------|------|
| 源字段名 | 数据库中的原始字段名 | order_no |
| 别名 | 业务显示名 | 订单编号 |
| 描述 | AI 上下文理解 | 客户提交的订单唯一标识 |
| 显示规则 | 值域转换 | `{"0":"未支付","1":"已支付"}` |

## 6. 权限管理

系统采用 RBAC（基于角色的访问控制）模型。

### 6.1 用户管理

**用户列表:**
- 查看所有系统用户
- 支持按用户名搜索

**创建用户:**
1. 进入「用户管理」
2. 点击「新增用户」
3. 填写：用户名、密码、邮箱
4. 选择用户角色
5. 点击「保存」

**分配角色:**
1. 点击用户操作列的「分配角色」
2. 在弹窗中勾选要分配的角色
3. 点击「确定」

### 6.2 角色管理

**创建角色:**
1. 进入「角色管理」
2. 点击「新增角色」
3. 填写：角色名称、角色代码、描述
4. 点击「保存」

**分配菜单权限:**
1. 点击角色操作列的「分配菜单」
2. 在菜单树中勾选该角色可访问的菜单
3. 点击「确定」

### 6.3 菜单管理

菜单管理用于维护系统的功能入口。

**创建菜单:**
1. 进入「菜单管理」
2. 点击「新增菜单」
3. 填写：
   - 菜单名称
   - 路由路径
   - 图标选择
   - 上级菜单（用于构建层级）
   - 排序号

## 7. 提示词规则

提示词规则用于优化 AI 查询的准确性和上下文理解。

### 7.1 创建提示词规则

1. 进入「提示词」→「提示词规则」
2. 点击「新增规则」
3. 填写：
   - **规则名称**: 规则描述性名称
   - **匹配关键词**: 触发该规则的关键词（多个用逗号分隔）
   - **系统提示词**: 要注入到 AI 的额外上下文
4. 点击「保存」

### 7.2 规则示例

| 规则名称 | 匹配关键词 | 系统提示词 |
|----------|----------|------------|
| 销售数据分析 | 销售,营收,收入 | 你是一个专业的销售数据分析助手... |
| 库存管理查询 | 库存,入库,出库 | 你是一个专业的仓储管理人员... |

## 常见问题

### Q1: 登录失败怎么办？
- 确认用户名密码是否正确
- 确认是否已联系管理员创建账号

### Q2: AI 查询返回空结果
- 检查自然语言描述是否清晰
- 确认数据源连接正常
- 检查表映射和字段映射配置

### Q3: 数据源连接失败
- 确认数据库服务正在运行
- 检查主机地址和端口是否正确
- 确认用户名密码有访问权限

### Q4: 如何导出查询结果？
- 在查询结果页面，点击工具栏的「导出 PDF」按钮
