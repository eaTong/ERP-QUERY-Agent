# 使用指南

## 快速开始

### 1. 登录系统

访问 http://localhost:3000 ，使用默认账号登录：

- 用户名: `admin`
- 密码: `admin123`

### 2. 配置数据源

进入 **数据源 > 外部数据源**，点击「添加数据源」：

1. 填写数据源名称
2. 选择数据库类型 (MySQL/SQL Server)
3. 填写主机地址、端口、数据库名
4. 填写用户名和密码
5. 点击「测试连接」确认连接正常
6. 保存

### 3. 配置表映射

进入 **数据源 > 表映射**，点击「添加映射」：

1. 选择已配置的数据源
2. 选择要映射的外部表
3. 填写本地别名（用于界面显示）
4. 填写使用场景描述
5. 填写查询规则（如 `isDelete=0`）
6. 保存

### 4. 配置字段映射

在表映射列表中，点击「字段映射」：

1. 同步外部字段（点击「同步外部字段」按钮）
2. 为每个字段设置本地别名
3. 设置显示规则（如 `{"0":"保存","1":"提交"}`）
4. 保存

### 5. 配置提示词规则（可选）

进入 **数据源 > 提示词规则**，点击「添加规则」：

1. 填写规则名称
2. 填写规则简介
3. 填写完整的提示词内容
4. 启用规则

### 6. 使用 AI 查询

进入 **AI 查询** 页面：

1. 在输入框中用自然语言描述查询需求
2. 点击「查询」按钮
3. 查看返回的查询结果

## 最佳实践

### 1. 查询规则

在表映射中设置查询规则可以简化 AI 查询：

```
isDelete=0  -- 只查询未删除的数据
status=1     -- 只查询已启用的数据
```

### 2. 显示规则

使用显示规则可以将数据库中的状态值转换为可读的文本：

```json
{"0": "保存", "1": "提交", "2": "审核通过", "3": "审核拒绝"}
```

### 3. 提示词规则

编写清晰的提示词可以提高 AI 理解准确性：

```
你是一个专业的ERP数据查询助手。当用户询问销售订单时：
1. 优先使用 orders 表（本地别名：销售订单）
2. 注意应用默认的查询条件 isDelete=0
3. status=1 表示已提交，status=2 表示已完成
4. 返回结果时使用中文显示
```
